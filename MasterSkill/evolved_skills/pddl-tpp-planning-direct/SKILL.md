# PDDL TPP Planner Direct Solver

Use this skill for `pddl-tpp-planning` and similar tasks where the judge expects the exact
plan generated by `unified_planning` with `OneshotPlanner(name="pyperplan")`.

## When to Use

- The task provides `problem.json` entries with `domain`, `problem`, and `plan_output`.
- The environment already includes `unified_planning`, `up-pyperplan`, and `pickle`.
- The verifier compares the saved `.pkl` plan object and the `.txt` action list against a
  fresh `pyperplan` run.

## How to Use

1. Do not hand-write plans. Recompute them with `PDDLReader` and `OneshotPlanner(name="pyperplan")`.
2. For every entry in `/app/problem.json`, parse the PDDL domain and problem exactly as listed.
3. Save the returned sequential plan in two forms:
   - `<plan_output>`: one `str(action)` per line
   - `<plan_output>.replace(".txt", ".pkl")`: `pickle.dump(plan, file)`
4. Fix `PYTHONHASHSEED` to a constant before solving so the later verifier process uses the
   same `pyperplan` tie-breaking order.
5. Validate the plan with `SequentialPlanValidator` before writing success.
6. If a plan cannot be found or validation fails, stop and surface the error rather than
   writing a guessed answer.

## Suggested Command

Run the deterministic wrapper inside the task container:

```bash
bash /root/.skills/pddl-tpp-planning-direct/scripts/run_deterministic_solver.sh
```

If the task environment uses another agent skill directory, the same script is also copied
under `/root/.codex/skills/pddl-tpp-planning-direct/scripts/solve_tpp_batch.py`.
